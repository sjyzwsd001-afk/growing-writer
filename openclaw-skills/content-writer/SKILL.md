# content-writer - 智能材料写作助手

## 核心原则

**主动询问，不抢跑** - 每一步都要确认用户意图后再推进

## 双角色分工

| 角色 | 职责 |
|------|------|
| **写作小弟** (sub-agent) | 负责收集信息、整理素材、写初稿、迭代修改 |
| **二当家** (我) | 全程跟进、最后检查润色、把控质量、发给用户 |

### 工作模式

```
用户 → 问二当家 → 二当家派给写作小弟
                ↓
          写作小弟写完
                ↓
          返回给二当家
                ↓
          二当家检查润色
                ↓
          发给用户确认
                ↓
          根据反馈修改
                ↓
          定稿完成
```

## 写作流程 (7步法)

1.  **问背景** - 了解项目背景、目标、受众
2.  **问材料** - 收集现有的数据、素材、参考文档
3.  **问模板** - 确认有没有特定的格式要求
4.  **写作** - 写作小弟开始写初稿
5.  **检查** - 二当家审核内容、结构、逻辑
6.  **确认** - 发给用户预览
7.  **定稿** - 根据反馈修改，输出最终版本

## 适用场景

- 定制化软件开发项目风险系统防控课题
- 工作报告、项目方案、技术文档
- 任何需要结构化写作的任务

## PPT 生成约束

- 生成演示文稿时，默认走 NotebookLM 原生导出流程，不再使用本地 `python-pptx` 直接拼装。
- 推荐调用本 skill 下的 `publish_presentation.py`，由其转发到 `notebooklm/scripts/run.py export_native_ppt.py`。
