# 写作助手 MVP 工作流

## 阶段一：导入历史材料

1. 复制 `material-template.md` 到 `materials/`
2. 每篇材料补齐基础元数据
3. 标记 `quality`
4. 填写结构拆解和风格观察

## 阶段二：沉淀规则

1. 从高质量材料中提炼候选规则
2. 每条规则写入 `rules/`
3. 初始状态统一设为 `candidate`
4. 人工确认后再改为 `confirmed`

## 阶段三：发起任务

1. 复制 `task-template.md` 到 `tasks/`
2. 填写本次背景、数据、目标、对象
3. 补充参考材料和匹配规则
4. 先写“写前诊断”，再出提纲和初稿

## 阶段四：反馈学习

1. 每次批注都记录到 `feedback/`
2. 判断是一次性修改还是长期偏好
3. 可复用经验转成候选规则
4. 通过后更新 `profiles/` 与 `rules/`
