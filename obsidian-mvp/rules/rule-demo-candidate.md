# 规则内容

风险类材料在描述问题时，不能只停留在原则判断，需要明确影响范围，并在措施部分给出可执行动作。

# 规则分类

- 类型：逻辑 / 信息完整性
- 适用范围：风险汇报
- 优先级：medium

# 来源依据

- 来自哪些历史材料：待补充
- 来自哪些反馈记录：feedback-demo
- 典型例子：问题分析太虚，措施只写原则

# 触发条件

- 什么时候应该使用这条规则：面向领导的风险说明或阶段汇报
- 哪些场景不适用：仅做简短状态同步时

# 正反例

## 正例

- 明确说明影响模块、进度影响和下一步动作

## 反例

- 只写“存在一定风险，后续持续跟进”
