# 写作助手功能方案

## 1. 功能目标

做一个“可学习的材料写作助手”。

它不是简单套模板，而是：

1. 先读取你过去写过的材料，学习你的写法、结构、常用表达和判断习惯。
2. 再接收新的原始素材，按你习惯的方式生成新材料。
3. 你指出哪里不对后，它能记住这类纠错，后面逐步写得更贴近你的要求。

## 2. 用户使用流程

### 第一步：喂历史材料

你上传或粘贴：

- 以前写过的材料
- 不同类型的范文
- 被采纳/被通过的正式版本
- 你自己总结过的写作要求

系统自动做三件事：

1. 拆结构：识别开头怎么写、正文怎么展开、结尾怎么收。
2. 抽规则：总结语气、篇幅、常见逻辑、禁忌表达、重点因素。
3. 建风格档案：形成你的“写作画像”。

### 第二步：输入新素材

你再提供：

- 新项目背景
- 新数据、新事实、新要求
- 目标场景
- 这次写给谁看

系统先判断缺什么信息，再提示你补充，然后开始写。

### 第三步：生成初稿

生成时不是盲写，而是基于：

- 历史材料学到的套路
- 当前任务的素材
- 已沉淀的规则
- 当前文体模板

输出内容最好分成两个层级：

1. 正式稿
2. 写作依据说明（可选展开）

这样你能知道它为什么这么写。

### 第四步：人工纠错

你可以直接批注：

- 这段太空
- 这里要更正式
- 这个顺序不对
- 这一类风险要单独写
- 结尾必须落到措施，不要只写问题

系统把反馈分成三类：

1. 本次修改
2. 可复用规则
3. 个性偏好

其中第 2、3 类会进入长期记忆。

### 第五步：持续优化

后续再写类似材料时，系统优先参考：

- 你的历史材料
- 你确认过的规则
- 你最近几次的纠错记录

这样会越写越像你，而不是每次重新开始。

## 3. 核心能力设计

### 3.1 材料学习能力

系统需要具备：

- 文档解析：支持 Word、PDF、Markdown、纯文本
- 分段识别：识别标题、层级、段落关系
- 类型识别：判断是汇报、方案、总结、申请、说明还是别的文体
- 风格归纳：提炼常见句式、行文节奏、专业词、禁用词

### 3.2 规则沉淀能力

要把“你觉得对/不对”变成明确规则，而不是只改当前一稿。

规则示例：

- 开头先写背景，再写目标
- 不要出现口语化表达
- 风险类材料必须包含原因、影响、措施
- 面向领导的版本不要展开技术细节
- 结尾必须有态度表述和下一步安排

建议把规则分为：

1. 通用规则
2. 文体规则
3. 个人偏好规则
4. 场景规则

### 3.3 写作生成能力

生成时应分阶段进行：

1. 先列提纲
2. 再生成全文
3. 再做语言统一和逻辑检查

这样比一次性出全文更稳。

### 3.4 反馈学习能力

你每次修改后，系统要能判断：

- 这是一次性的改动，还是长期规则
- 这是措辞问题，还是结构问题
- 这是事实修正，还是表达偏好

否则它会“记错东西”。

## 4. 推荐的功能模块

### 模块一：材料库

用途：存放历史材料。

建议支持：

- 上传材料
- 给材料打标签
- 标记“高质量样本”
- 按类型分类
- 搜索历史段落

### 模块二：规则库

用途：沉淀系统学到的写作规则。

每条规则最好包含：

- 规则内容
- 来源材料
- 适用场景
- 置信度
- 最近一次被验证时间

### 模块三：任务面板

用途：每次发起新的写作任务。

字段建议：

- 写作目标
- 文体类型
- 面向对象
- 本次素材
- 特殊要求
- 篇幅要求
- 截止时间

### 模块四：生成与修改面板

用途：展示初稿和修改痕迹。

建议支持：

- 生成提纲
- 生成全文
- 逐段重写
- 对比修改前后
- 记录用户批注
- 一键沉淀为长期规则

### 模块五：记忆中心

用途：管理“系统学会了什么”。

要能看到：

- 已学习的风格特征
- 已沉淀的规则
- 最近新增的纠错经验
- 可删除或禁用的错误记忆

## 5. 最关键的产品逻辑

这个功能真正的核心不是“生成”，而是“可控学习”。

如果只会读历史材料然后模仿，它很容易：

- 学到表面语气，学不到判断逻辑
- 把旧材料里的过时表达继续沿用
- 把一次性改动误认为长期规则

所以必须有一层“规则提炼 + 人工确认”机制。

推荐逻辑：

1. 系统从历史材料里自动总结规则。
2. 由你确认哪些规则保留。
3. 新写作任务优先调用已确认规则。
4. 你修改后，再把反馈转成候选规则。
5. 规则不断更新，写作质量才会稳定提升。

## 6. 一个最小可用版本（MVP）

如果先做第一版，不要一开始就做得太重。

MVP 只做 4 件事：

1. 导入 10-30 篇历史材料
2. 自动总结写作风格和规则
3. 输入新素材后生成初稿
4. 用户批注后把反馈沉淀成规则

第一版先不追求：

- 非常复杂的自动分类
- 完整知识图谱
- 全自动长期记忆纠偏

先把“能学、能写、能改、能记住”跑通。

## 7. 适合你的交互方式

如果你是高频写材料的人，最顺手的交互不是纯聊天，而是“任务表单 + 可编辑草稿 + 批注反馈”。

建议采用：

1. 左侧：本次任务信息和素材
2. 中间：系统生成的大纲/正文
3. 右侧：已学习规则和历史参考依据

这样你会更容易控制结果。

## 8. 后续可增强的方向

- 区分不同对象的写法，比如领导版、专家版、客户版
- 自动识别你常遗漏的因素，提前提醒
- 从你修改最多的地方反推你的写作偏好
- 针对不同材料类型建立不同写作代理
- 支持“基于某篇历史材料的风格仿写”

## 9. 一句话定义

这是一个“能从你的历史材料中学会写作套路，并通过你的持续纠错不断进化的个性化写作助手”。
